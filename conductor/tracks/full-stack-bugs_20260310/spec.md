## Problem Description

### P0 — Critical (blocks CI/CD)

1. **`types-redis==4.6.0.12` does not exist on PyPI** — This version was pinned in `backend/requirements-dev.txt` but PyPI switched to date-based versioning (`4.6.0.20241004`+). This breaks ALL E2E test runs and staging deploys that install dev dependencies.
   - Steps: Run `pip install types-redis==4.6.0.12` → `ERROR: No matching distribution found`
   - Fix: Update to `types-redis>=4.6.0.20241004`

2. **`/internal/fetch-weather` returning HTTP 422** — The fetch-weather cron workflow has 10+ consecutive failures. The endpoint returns 422 (Unprocessable Entity), suggesting a request body/parameter validation error.
   - Steps: Check GHA `fetch-weather.yml` run history → all recent runs fail with 422
   - Root cause: Needs investigation — likely missing required field or schema mismatch

### P1 — High

3. **Gitleaks detected secret leaks in commit `26cd606`** — The secret-scan workflow found potential credential exposure. SARIF report needs review to determine severity and whether rotation is needed.

4. **Missing `self-healing` and `automated` GitHub labels** — The self-healing-monitor workflow creates issues with these labels, but the labels don't exist in the repo, causing label assignment failures.
   - Fix: `gh label create self-healing` and `gh label create automated`

### P2 — Medium

5. **CSP blocks Clarity analytics** — `scripts.clarity.ms` is blocked by the Content Security Policy. The `script-src` directive includes `https://www.clarity.ms` but Clarity loads scripts from `scripts.clarity.ms` (subdomain mismatch). Also `img-src` doesn't include `c.clarity.ms`.
   - Fix: Update CSP in `next.config.js` to include `https://*.clarity.ms` in `script-src` and `img-src`

6. **Missing HTTP security response headers on Vercel** — x-frame-options, x-content-type-options, strict-transport-security, and content-security-policy headers are not returned as HTTP headers, even though CSP is configured in next.config.js. Vercel may need explicit `headers()` config.
   - Fix: Add security headers in `next.config.js` `headers()` function or `vercel.json`

7. **3 cron workflows with Slack notification failures** — fetch-weather, market-research, and scrape-rates had empty `SLACK_INCIDENTS_WEBHOOK_URL` (fixed in previous session but may need verification).

8. **`data-retention.yml` and `model-retrain.yml` may not have run** — Schedule-only workflows with no recent successful runs; may need manual trigger or schedule verification.

### P3 — Low (informational)

9. **Frontend Jest: 3 pre-existing failures in `send.test.ts`** — Known issue, email test mocking gaps. Not a regression.

10. **E2E tests require local backend** — E2E Playwright tests fail with ECONNREFUSED when no local backend is running at localhost:8000. This is expected behavior, not a bug — E2E tests run in CI with both servers.

11. **Console 401 error on homepage** — An unauthenticated API call returns 401. Expected for non-logged-in users but could be suppressed.
