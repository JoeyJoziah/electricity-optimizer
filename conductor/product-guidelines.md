# Product Guidelines

## Voice and Tone

**Friendly and approachable.** Use warm, helpful language that makes energy concepts accessible to everyday consumers. Avoid jargon unless necessary -- when technical terms are required, explain them inline.

### Examples

- **Do:** "You could save $12/month by switching to off-peak usage hours."
- **Don't:** "Temporal arbitrage of TOU rate differentials yields 8.3% cost reduction."
- **Do:** "We'll keep an eye on prices and let you know when they drop."
- **Don't:** "Automated threshold-based alerting has been configured."

## Design Principles

### 1. Actionable Insights

Every screen, notification, and data point should drive a clear action. If a user sees a chart, they should know what to do next. Avoid information-only displays -- always pair data with a recommendation.

### 2. Seamless Integrations

Utility connections should "just work." Minimize the number of steps to connect an account. Handle errors gracefully with clear recovery paths. Never leave a user stranded in a broken connection state.

### 3. Trust Through Transparency

Users should see how savings are calculated. Show your work -- display the data sources, the methodology, and the confidence level. Never present an estimate without explaining how it was derived.

## Accessibility

- WCAG 2.1 AA compliance minimum
- All interactive elements keyboard-navigable
- Screen reader support for all pages (jest-axe coverage: 19/19 pages)
